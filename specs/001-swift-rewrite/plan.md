# Implementation Plan: Full Swift Rewrite of Bitwarden Alfred Workflow

**Branch**: `001-swift-rewrite` | **Date**: 2026-05-29 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-swift-rewrite/spec.md`

## Summary

Rewrite the Bitwarden Alfred Workflow from scratch in Swift, compiling to a single
`bw-alfred` binary. Alfred invokes it with a command name as the first argument; the
binary dispatches, reads Alfred variables from the environment, and writes Alfred script
filter JSON to stdout. The vault is cached on disk (sensitive fields stripped), with all
reads served from cache and only explicit syncs or live TOTP fetches touching the network.
The `bw serve` background process is managed via PID file. The master password is stored
in the macOS Keychain for seamless auto-unlock.

## Technical Context

**Language/Version**: Swift 5.9, Swift Package Manager, single executable target

**Primary Dependencies**: Foundation (URLSession, Process, FileManager), Security.framework
(Keychain), CryptoKit (HMAC-SHA1 for TOTP), UserNotifications or `osascript` for
notifications, no third-party packages

**Storage**: JSON files on disk (vault cache + prefs + recency), macOS Keychain (master
password), PID file (bw serve lifecycle)

**Testing**: XCTest (`swift test`); unit tests for TOTP computation, Base32 decoding,
eTLD+1 matching, cache stripping, Alfred JSON encoding

**Target Platform**: macOS 12+ (Monterey); Swift 5.9 / Xcode 15

**Project Type**: CLI tool / Alfred workflow binary

**Performance Goals**: Item list rendered in < 500ms (cache-served); full workflow
(invoke → clipboard) in < 5 seconds

**Constraints**: Single binary, no bundled dependencies, `bw` CLI and `jq` are runtime
dependencies; no hardcoded home directory paths

**Scale/Scope**: Single-user desktop tool; ~25 commands; vault up to ~1000 items

## Key Design Decisions

### Command Dispatch (`main.swift`)

```swift
switch CommandLine.arguments.dropFirst().first ?? "" {
case "main":           MainMenu.run()
case "search":         Search.run()
case "get_field":      GetField.run()
// ... all ~25 commands
default:               AlfredOutput.error("Unknown command").print()
}
```

Each command is a static `run()` function. Arguments beyond the command name are accessed
via `CommandLine.arguments[2...]`. Environment variables via `ProcessInfo.processInfo.environment`.

### `ensureUnlocked()` Chain

Called at the top of every command that needs vault access:

1. Check PID file → if no PID or process not running → call `start_server`
2. `GET /status` → if `unauthenticated` → call `login` flow
3. `GET /status` → if `locked` → call `unlock` flow (Keychain → dialog fallback)
4. Proceed with command

### Sensitive Field Stripping (`CacheBuilder`)

`CacheBuilder.build(from: BWItem) -> CachedItem` explicitly zeros out:
- `login.password`
- `login.totp`
- `card.number`, `card.code`
- Any `fields` entry where `type == .hidden` → `value = nil`
- Sets `hasTOTP = (item.login?.totp != nil) || item.fields?.contains { $0.name?.lowercased() == "totp" && $0.type == .hidden } == true`

### Clipboard Management

`get_field` writes to clipboard via `NSPasteboard`, saves previous contents, then
schedules a `DispatchQueue.global().asyncAfter` to restore. The restore delay is
`bw_clipboard_time` seconds (default 30).

### Browser URL Retrieval

`URLMatcher.browserURL()` runs an AppleScript snippet via `Process` against the frontmost
browser application to get the current tab URL. Returns `nil` gracefully for unsupported
browsers or non-browser frontmost apps.

## Implementation Order

1. **Models** (`Alfred/`, `Cache/` types, `BWClient/` response types) — Codable structs,
   no logic
2. **BWServer** — PID file, `bw serve` start/stop, `ensureUnlocked()` skeleton
3. **BWClient** — URLSession sync wrappers for all `bw serve` endpoints
4. **Cache** — `CacheBuilder` (stripping), `VaultCache` read/write, `WorkflowPrefs`,
   `RecencyStore`
5. **Support modules** — `Keychain`, `TOTP`+`Base32`, `URLMatcher`, `Notifier`,
   `LaunchAgent`
6. **Commands** — one file per command, wiring all modules; `main.swift` dispatch last
7. **Tests** — unit tests for pure-logic modules (TOTP, Base32, URLMatcher, CacheBuilder,
   Alfred JSON encoding)
